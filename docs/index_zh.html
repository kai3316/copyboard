安全架构

纵深防御

传输加密 + 落盘加密，全程守护你的数据。

🔒
应用层

AES-256-GCM 逐帧加密

  • 密钥通过 HKDF 从 Ed25519 证书派生，每对设备独立
  • 可选预共享密码，进一步增强加密强度（PBKDF2，60 万次迭代）
➔
🔑
传输层

TLS 1.3 + Ed25519

  • Ed25519 自签名证书，端到端身份验证
  • 证书锁定 —— 首次信任后持续校验，身份变更立即预警
➔
💾
静态存储

AES-256-GCM 落盘加密

  • 配置文件和剪贴板历史均加密存储
  • 密码仅存储 PBKDF2 验证哈希（密码本身永不落盘）
工作原理

四步完成同步

从发现到粘贴，不到一秒。

1

发现

通过 mDNS 在局域网内自动发现其他设备。

2

配对

两端确认相同的 8 位配对码，锁定对方证书，建立长期信任。

3

同步

检测剪贴板变化 → 哈希去重 → 加密 → 发送 → 写入目标设备。

4

信任

已配对设备自动重连。证书一旦变更，立即发出安全预警。

常见问题

还有疑问？

不会。所有数据仅在局域网内点对点传输，TLS 1.3 加密保护。ClipSync 没有任何服务端组件，你的剪贴板数据永远不会离开你的本地网络。
支持。Windows、macOS、Linux 三个平台均可运行，不同系统之间无缝同步。Windows 上复制的内容可以直接在 MacBook 上粘贴。

ClipSync 采用双层加密：TLS 1.3 保护传输通道，AES-256-GCM 逐帧加密应用数据。首次配对使用 8 位验证码防止中间人攻击，之后启用证书锁定 —— 一旦设备证书发生变化，立即发出安全预警。
